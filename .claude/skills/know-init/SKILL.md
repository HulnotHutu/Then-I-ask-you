# know-init — 文档阅读 → knowledge.md

从原始文档中提取结构化知识，写入 `documents/knowledge.md`。

核心原则：**先读文档，再写知识库**。所有知识必须有来源，不能跳过文档直接编造。

## 工作台结构

```
workbench/
└── documents/
    ├── knowledge.md           # 提炼后的知识库（唯一输出）
    └── <原始文档>             # 待学习的参考文档
```

- 可以读取 `documents/*` 下所有文件
- 只能修改 `documents/knowledge.md`
- 如果 `workbench/` 或 `documents/` 目录不存在，先创建它们

## 工作流程

### 0. 获取文档

1. 从用户输入判断是路径引用还是内联文本
2. 如果是路径，用 Read 读取指定文件
3. 如果用户没有指定具体文件，用 Glob 列出 `documents/*`，跳过 `.gitkeep` 和 `knowledge.md` 自身
4. 多个文档逐一处理

### 1. 判断模式：初始化 or 认知补全

检查 `documents/knowledge.md` 是否存在，以及其中 **元信息 → 已学习文档列表** 是否包含当前文档路径。

**模式 A — 初始化**（文档不在已学习列表中）：
- `knowledge.md` 尚不存在，或当前文档未被学习过
- 执行完整的三遍阅读法（预览→速读→精读）
- 在 `knowledge.md` 中新增完整的 `## 文档路径` 章节

**模式 B — 认知补全**（文档已在已学习列表中）：
- 当前文档已被标记为 `[x]` 或 `[↻]`
- 用户希望重新深入阅读，或之前阅读有遗漏
- 重新精读文档（可精简预览/速读，重点放在精读上），关注之前未覆盖的段落或理解不够深入的部分
- 在已有章节末尾追加 `#### 补全笔记（yyyy-mm-dd）` 子节

### 2. 三遍阅读法

按 预览→速读→精读 顺序执行，不可跳步。

**预览** — 读标题、摘要/引言、各级小标题、结论、图表标题、参考文献标题。跳过方法/实验细节。产出：5-6 个关键词/核心术语，确定文档领域和章节骨架。

**速读** — 根据章节骨架，读每段第一句 + 最后一句。标注每段大意（一个词或短句）。问自己：这些段落讲了什么？能回答什么问题？将相关段落归入对应小标题。

**精读** — 只对难懂或关键部分深入。做笔记、写疑问、评估、批评。读完做到"标题 + 一句话核心贡献 + 两个最强证据 + 一个批评"。

**结尾** — 用三句话复述论证脉络：
- 问题：解决了什么实践问题？回答了什么问题？
- 方案：用了什么方法论？如何实现？
- 结果：达成了什么效果？产生了什么结论？

### 3. 写入 knowledge.md

**knowledge.md 不存在** → 创建新文件，写入元信息 + 完整文档章节。

**knowledge.md 存在但文档未学习（初始化）** → 在已学习文档列表中追加 `[x] 文档路径`，在文件末尾追加完整文档章节，更新全局关键词索引。

**文档已学习（补全）** → 将已学习文档列表中该条目更新为 `[↻] 文档路径（最近补全：yyyy-mm-dd）`，在已有章节末尾追加补全笔记，更新全局关键词索引。

## knowledge.md 模板

```markdown
# executor's perception

## 元信息
- 已学习文档列表：
  - [x] `文档路径1`
  - [↻] `文档路径2`（最近补全：2026-05-13）
- 知识领域：<领域名>
- 全局关键词索引：
  - 关键词1 → `文档路径1` §章节名
  - 关键词2 → `文档路径2` §章节名

---

## `文档路径1`

### 小标题 A（依据预览生成）

> **速读总结**（1-3句概括本小节核心内容）
>
> **可回答问题**：本节能回答什么类型的问题？

#### 精读笔记
- **关键概念**：概念名（`原文行Lxx-Lyy`）：解释
- **证据/数据**：`Lxx-Lyy`：引用原文关键句或数据
- **质疑/批评**：`Lxx-Lyy`：你的疑问或对原文局限性的判断
- **关联**：与其他小标题或外部文档的关系

### 小标题 B

> **速读总结**：...
>
> **可回答问题**：...

#### 精读笔记
- ...

#### 补全笔记（2026-05-13）
- **新理解**：之前遗漏或理解不深的内容
- **新质疑**：重新阅读后产生的新问题
- **新关联**：与其他章节或外部知识的新关联

### 三句话复述
1. 问题：解决了什么实践问题？回答了什么问题？
2. 方案：用了什么方法论？如何实现这个解决方案？
3. 结果：达成了什么效果？完成了什么指标？产生了什么结论？

---

## `文档路径2`

...
```

### 4. 完成后输出摘要

写入完成后，向用户报告：
- **本次操作**：初始化了哪个文档 / 补全了哪个文档
- **新增/更新章节**：新建了哪些小标题及其核心内容（1 句概括）
- **全局关键词索引**：当前所有关键词及其对应章节

### 5. 超长文档处理

- 先读前 3000 行，在元信息中记录已读行数
- 告知用户已读进度，询问是否继续
- 分阶段补全直到读完

## 约束

| 项目 | 规则 | 原因 |
| :--- | :--- | :--- |
| 知识来源 | 必须基于用户给出的文档，不能跳过文档直接写 `knowledge.md` | 保证知识有根有据 |
| 模式判断 | 先检查元信息中的已学习文档列表，再决定初始化还是补全 | 避免重复创建章节，支持渐进加深 |
| 追加策略 | `knowledge.md` 已存在时追加，不覆盖已有内容 | 知识可以累积 |
| 三遍阅读 | 必须按预览→速读→精读顺序执行 | 预览定骨架，速读填内容，精读深入关键点 |
| 元信息维护 | 始终更新已学习文档列表和全局关键词索引 | 方便后续检索定位 |
| 超长文档 | 先读前 3000 行，记录已读行数，分阶段补全 | 大文件单次无法完整处理 |
