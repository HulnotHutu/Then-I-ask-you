# 理解 Transformer

## 一句话抓住本质
Transformer 是一个放弃递回和卷积、完全依靠注意力机制来建模序列全局依赖的神经网络架构——它让序列中每个位置都能**直接、并行地**与所有其他位置交互。

---

## 第一步："它像什么？"

### 类比场景：圆桌会议 vs 流水线

想象一个团队要共同理解一份长文件，有两种方式：

**RNN 的方式——流水线传话**：
- 大家排成一列。第一个人读完第一段，写下摘要传给第二个人；第二个人结合自己的段落和收到的摘要，更新后传给第三个人……
- 每个人必须等前面的人完成才能开始。坐在末尾的人收到的摘要经过多轮传递，早就模糊了。
- 座次相隔很远的人无法直接沟通，信息只能一站一站地传。

**Transformer 的方式——圆桌会议**：
- 所有人围坐在圆桌前。每个人站起来向全场说出自己的内容，同时可以**同时听取所有人**的发言。
- 你可以决定重点听谁说的（比如权威的同事权重更高），也可以忽略无关的。
- 每个人说完一轮后，可以基于刚才听到的所有内容再说第二轮，此时你的发言已经融合了全场的信息。
- 整个过程**没有先后顺序的等待**，所有人同时发言、同时收听。
- 桌上有块牌子写着每个人的发言序号，帮你理解"这是谁、在第几位发言的"。

### 对应关系

- **每个与会者** ⇔ **序列中的一个词/位置**
- **向全场发言** ⇔ Self-Attention 中的 **query**（我要问什么）
- **听取所有人的发言** ⇔ Self-Attention 中每个位置关注所有位置的 **key-value** 对
- **决定重点听谁** ⇔ **注意力权重**（softmax 加权）
- **从语气、表情、内容等多个角度听** ⇔ **Multi-Head Attention**（多个表示子空间）
- **发言顺序牌** ⇔ **Positional Encoding**（注入位置信息）
- **第一轮所有人都说完** ⇔ **Encoder 的一层**（所有位置同步更新）
- **基于第一轮的内容再说第二轮** ⇔ **Encoder 堆叠多层**（信息逐步抽象）
- **你说出最终结论时，每说一个字都参考之前已经说过的字** ⇔ **Decoder 的自回归 + masked self-attention**

## 第二步："但它不完全是……"

| 你可能会觉得… | 实际上的精确真相是… |
| :--- | :--- |
| Self-Attention 就是算两个词之间的相似度 | 实际上是 **query 和 key 的点积 + softmax 得到权重，再对 value 做加权平均**。query 和 key 来自不同的线性投影，不是"词本身"直接做相似度。 |
| Attention 能自动理解语义 | 注意力机制只是数学运算（加权平均），**本身没有语义能力**。语义来自训练中学到的 projection 矩阵（W^Q, W^K, W^V）。"理解"是数据驱动的结果，不是 attention 的内在属性。 |
| Transformer 没有顺序信息（因为是并行计算） | 有，通过 **Positional Encoding** 显式注入。但它**加到 embedding 上而不是拼接**，意味着位置信息可能被语义信息掩盖。这是一个设计权衡，不是完美的方案。 |
| 多头注意力就是把 attention 重复算 8 次 | 每个头有不同的 **线性投影**，将 Q/K/V 映射到不同的表示子空间，捕获不同类型的依赖关系（有的是语法关系，有的是语义相关性）。这是通过 **参数不同** 实现的，不是简单重复。 |
| Transformer 的计算量比 RNN 小 | 当序列长度 n **小于** 表示维度 d 时才是。当序列很长时，Self-Attention 的复杂度是 **O(n²·d)**，这会急剧增长。这也是为什么后来出现稀疏注意力、线性注意力等变体。 |
| 残差连接只是让网络更深的手段 | 在 Transformer 中残差连接还有一个关键作用：**使"LayerNorm(x + Sublayer(x))"成立的必要前提**——两个子层的输出维度必须一致（d_model=512），残差才能相加。这约束了整个架构设计。 |

> **一句话**：Transformer 用"全场直连 + 多头加权"代替了"逐站传递"的序列建模方式，让你看到全局依赖可以靠纯注意力高效捕获，本质上是**用平方级别的计算量换取常数级别的信息路径长度和完全的并行化**。

## 第三步："它到底是什么？"

### 专业定义

> Transformer 是一种基于纯注意力机制的序列转录模型，由 Vaswani et al. (2017) 在 "Attention Is All You Need" 中提出。它通过堆叠的多头自注意力层和位置前馈网络构建 encoder 和 decoder，完全摒弃递回和卷积，在机器翻译任务上达到当时的最高水平。根据 Google Scholar，截至 2026 年该论文引用量已超过 15 万次。

### 架构拆解

- **"Scaled Dot-Product Attention"** — `Attention(Q,K,V) = softmax(QK^T / √d_k) V`。query 与所有 key 计算点积，除以 √d_k 防止梯度消失，经 softmax 得到权重后对 value 加权求和。除以 √d_k 的原因是：当 d_k 较大时，点积的方差 = d_k，会使 softmax 进入梯度极小的区域。
- **"Multi-Head Attention"** — 将 Q、K、V 分别投影 h 次（h=8，每个投影维度 d_model/h=64），并行计算 attention，拼接后再投影。让模型从不同的表示子空间同时关注信息。总计算量与单头全维度 attention 相近。
- **"Encoder"** — 6 层堆叠。每层：Multi-Head Self-Attention → Position-wise FFN。每个子层后跟残差连接 + Layer Normalization。
- **"Decoder"** — 6 层堆叠。每层：Masked Multi-Head Self-Attention（防止看到未来信息）→ Cross-Attention（关注 encoder 输出）→ FFN。同样残差连接 + Layer Normalization。
- **"Positional Encoding"** — 正弦/余弦函数 `PE(pos, 2i) = sin(pos/10000^(2i/d_model))`，与 embedding 相加。利用三角恒等式，任意位置偏移 k 的编码可表示为原位置的线性函数，便于模型学习相对位置关系。
- **"Position-wise FFN"** — `FFN(x) = max(0, xW₁ + b₁)W₂ + b₂`，相当于两层 1x1 卷积。输入输出维度 512，中间层 2048。

### 为什么要有这个架构？

**设计动机**：在 Transformer 之前，序列建模存在一个三重困境——

1. **RNN 的串行约束**：必须按时间步顺序计算，无法利用 GPU 并行能力。序列越长，训练越慢。
2. **CNN 的远距离衰减**：卷积核的局部感受野导致远距离位置间的信息需要经过多层传递，路径长度随距离增长。
3. **Attention 作为附属品**：Attention 已被证明有效，但始终挂在 RNN/CNN 之上，没被当作主力。

Transformer 的创新在于：**不是给现有架构加 attention，而是让 attention 从头到尾成为唯一的主力**。这意味着：
- 任意两个位置间的信息路径长度恒为 O(1)（一层即可直达）
- 所有位置的计算可以完全并行（O(1) 顺序步骤）
- 多头机制让同一个词在不同语义空间扮演不同角色

### 为什么它能成功？

从实验结果来看：
- **WMT 2014 英德翻译**：28.4 BLEU（big model），超越当时所有模型（含集成模型）2+ 个点。训练：3.5 天 / 8 张 P100。
- **WMT 2014 英法翻译**：41.8 BLEU，训练成本不到之前 SOTA 的 1/4。
- **消融实验证实**：注意力头数、模型尺寸、dropout 都是敏感参数；而位置编码的设计选择（正弦 vs 可学习）影响不大。

## 常见误解快查
- ❌ "Attention 是 Transformer 的全部" → ✅ FFN 占了 Transformer 约 **2/3 的参数**，attention 不是唯一的计算组件。
- ❌ "Transformer 学不到局部模式" → ✅ Position-wise FFN 加上残差连接，事实上可以捕获局部模式，只不过通过 FFN 而非卷积。
- ❌ "位置编码就是加一个序号" → ✅ 正弦编码的关键是让模型**易于学习相对位置**，因为 PE(pos+k) 可表示为 PE(pos) 的线性函数，而非只是唯一的绝对标识。
- ❌ "Transformer 不适用于长文本" → ✅ 原始 Transformer 的 O(n²) attention 在极长序列上确实代价高，但这是 scope 问题——后续的稀疏/线性/滑动窗口 attention 已大幅缓解了这一点。
